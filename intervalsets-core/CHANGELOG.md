# Changelog — intervalsets-core

All notable changes to the `intervalsets-core` crate are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

This crate ships in lockstep with `intervalsets`; both share the workspace
version and are released together via `cargo-release`. See the repo
[CONTRIBUTING.md](../CONTRIBUTING.md) for the versioning policy.

## [Unreleased]

### Added

- `enum_interval!` proc-macro — compile-time-checked literal for `EnumInterval<T>` that accepts the same grammar as the runtime `FromStr` impl. Re-exported from the crate root and `prelude`. Malformed input (bracket mismatch, missing comma, closed bracket on an unbounded side, set notation, crossed numeric-literal bounds) fails to **build** instead of panicking at runtime. Bound bodies are tokenized as Rust source so non-literal bounds (identifiers, expressions like `BigDecimal::from(0)`) work alongside numeric literals. An optional second argument supplies a storage-type hint emitted as a turbofish on the constructor — useful when there's no T-bearing argument to infer from (`enum_interval!("(.., ..)", i32)`), and additionally wraps every bound expression in `<T as From<_>>::from(...)` so any `From`-convertible source can be used as a bound (e.g. `enum_interval!("[0_i32, 10_i32]", f64)`, or a user newtype `Unit(f64)` with `impl From<f64> for Unit`). UFCS form is used so an inherent `from` method on the user's type can't shadow the trait method. The macro lives in a new `intervalsets-macros` workspace member; depends on `proc-macro2`/`quote`/`syn`/`proc-macro-crate`. Trybuild fixtures gate compile-error messages behind `INTERVALSETS_UI_TESTS=1` (run via `just test-ui`) to insulate the default workspace test from rustc diagnostic drift.
- `FromStr` impls for `FiniteInterval<T>`, `HalfInterval<T>`, and `EnumInterval<T>` — math-notation parser that round-trips against `Display`. Accepts `[a, b]` / `(a, b)` / `[a, b)` / `(a, b]` for finite, `[a, ..)` / `(a, ..)` / `(.., b]` / `(.., b)` for half-bounded, `(.., ..)` for unbounded, and `{}` for empty. Whitespace around delimiters and the comma is lenient. Bound: `T: Element + FromStr`. The `FiniteInterval` / `HalfInterval` impls reject inputs that parse to the wrong variant.
- `FromStr` impl for `MaybeDisjoint<T>` — accepts the same grammar as `IntervalSet` per `docs/specs/string_repr.md`: a bare §2 interval form (`{}`, `[0, 5]`, `(.., ..)`, …) becomes a `Connected` value; a brace-wrapped `{piece U piece U ...}` form is folded left-to-right with normalization (empties dropped, overlaps and adjacencies merged). Parsing fails with `ParseIntervalError::Syntax` when the running fold would exceed two disjoint pieces — inputs whose 3+ pieces collapse to ≤2 under merging are accepted. No alloc required: the fold streams over segments via a one-pass top-level `U`-splitter and the new `MaybeDisjoint::try_merge_interval` method.
- `MaybeDisjoint<T>::try_merge_interval(self, piece: EnumInterval<T>) -> Result<Self, (Self, EnumInterval<T>)>` — set-theoretic union with a single interval, restricted to the two-piece capacity. Returns `Err((self, piece))` (recovering the original inputs without a defensive clone) when `piece` is disjoint from both stored pieces of a two-piece `Disjoint` value (the union would require three disjoint pieces). Generalizes `MergeConnected` for `MaybeDisjoint`: where `MergeConnected` returns the merged single piece only when everything collapses (all-or-nothing), `try_merge_interval` also handles partial overlap by extending the connected stored piece while preserving the disjoint one. Bound: `T: Element`.
- `error::ParseIntervalError<E>` — parametric error returned by the new `FromStr` impls. Variants: `Syntax` (grammar failure), `Element(E)` (carries `T::FromStr::Err` verbatim), `InvalidElement` (mirrors `Error::InvalidElement`), `InvalidBoundPair` (mirrors `Error::InvalidBoundPair`). `From<Error>` lifts factory errors into the corresponding variants. Re-exported from the prelude.
- `ops::Bisect<T>` trait + `ops::Bisection<T, S>` result struct, with impls for `FiniteInterval`, `HalfInterval`, `EnumInterval`, and `MaybeDisjoint`. Single method `bisect(&self, closed: Side) -> Option<Bisection<T, Self>>` — bisection uses the natural `Measure` of `T` (cardinality on discrete, Lebesgue width on continuous). Single connected intervals split at the bound midpoint in one step (uniform measure density). Multi-piece sets walk the convex hull's value range, splitting at the bracket midpoint and walking toward the heavier half until the midpoint stabilizes (ulp-stable for floats, neighbor-stable for integers). Returns `None` for empty / unbounded inputs. `HalfInterval`'s impl is trivially `None`. Bounds: `T: Element + Clone + Midpointable<Error = Infallible>` (covers every std primitive numeric type plus `OrderedFloat` / `NotNan` / `BigInt` / `BigUint` / `BigDecimal` / `Fixed*`; excludes `Decimal` whose `Midpointable::Error` is `MathError`). The algorithm helper `bisect_core` is exposed as `#[doc(hidden)] pub` so the outer `intervalsets` crate can drive the same loop for `Interval` / `IntervalSet`.
- `core::num::Saturating<T>` is now a valid storage type — usable wherever a bare primitive integer was before. Every `Try*` op for `Saturating<T>` is `Error = Infallible`: saturation is *defined behavior* on the bounded integer lattice (closed, monotonic, ordering-preserving), so no overflow path needs to surface as `Err`. `core::num::Wrapping<T>` is intentionally NOT supported and will not be added — wrapping overflow flips ordering (`i8::MAX + 1 = i8::MIN`), violating the bound-pair invariant.
  - `Element` and `Midpointable` are blanket-impl'd over the matching capability trait on the inner `T`. `Saturating<T>` inherits the inner `T`'s `Element::Kind` and `Element::Measure` (e.g. `Saturating<i32>` is discrete with `Measure = u64` under stepwise widening).
  - `TryAdd` / `TrySub` / `TryMul` delegate to `T::saturating_*` (via `num_traits::Saturating{Add,Sub,Mul}` bounds).
  - `TryDiv` saturates by sign of the true result. Signed `MIN/-1` → `MAX` (matching stdlib's `core::num::Saturating<iN>::Div`). `_/0` is treated as a limit: `+n/0 → MAX`, `-n/0 → MIN`. `0/0` is mathematically indeterminate and falls out as `MAX` from the sign rule. For inner types where `|rhs|` can be less than 1 (fixed-point, decimal) the sign-of-result rule covers both overflow directions uniformly.
  - Cast surface: `TryCast` + `LossyCast` for reflexive (`Saturating<T> → Saturating<T>`), lift (`T → Saturating<T>`), and unwrap (`Saturating<T> → T`). **No Tier-1 `Cast`** — lifting doesn't change values but does change downstream arithmetic semantics, so it's an explicit opt-in via `TryCast`/`LossyCast`. Cross-`Saturating<T> ↔ Saturating<U>` and cross-with-primitive casts are deferred to a follow-up.
  - Orphan-rule limitation: `num_traits::{Bounded, Zero, One}` cannot be blanket-impl'd for `Saturating<T>` from this crate (both the trait and the type are foreign), so `IntoFiniteInterval` and `FiniteInterval<Saturating<T>>: Zero/One` are unavailable. Workaround: construct the `FiniteInterval` directly with `Saturating(T::MIN)` / `Saturating(T::MAX)`. The natural saturating-arithmetic usage pattern never escapes `FiniteInterval` to begin with — saturation keeps results in-range — so this is rarely felt in practice.
- `From<Infallible> for Error` — lets a storage type's `TryAdd`/`TrySub`/`TryMul`/`TryDiv` impl with `Error = Infallible` satisfy the set-level `<T as TryAdd>::Error: Into<Error>` bound without an intermediate `MathError` hop (which Rust's `From` doesn't compose). A latent gap that also affected `BigInt`/`BigUint` set-level arithmetic.
- `cast` module — storage-type cast trait family for converting the element type of a set without manual reconstruction:
  - `Cast<U>` (Tier 1, infallible) keyed on a new element-layer `CastElement<U>` trait. Primitive pairs are covered by a single blanket impl over `T: Into<U>` gated on the sealed `Primitive` marker; feature-gated storage types provide their own `CastElement` impls. Implemented on `FiniteBound`, `FiniteInterval`, `HalfInterval`, `EnumInterval`. The `CastElement` contract is "infallible by precondition": impls may assume the input is a valid bound limit for `U` (the set-level `Cast` impl always satisfies this — the input came from a `FiniteBound<T>` that already passed `T::validate`), enabling exact conversions like `f64 → BigDecimal` to be `.expect()`-internal at the element layer rather than `Result`-typed at the set layer.
  - `LossyCast<U>` (Tier 1, total but lossy) for pairs where the element conversion saturates / rounds. Out-of-range elements clamp to `U`'s extrema; bounds at saturation extrema snap to `Closed`. `FiniteInterval::lossy_cast` routes through `try_satisfy_bounds`, collapsing crossed bounds to `Empty`. Element layer is keyed on `LossyCastElement<U>`, with macro-generated impls covering every primitive pair (int↔int via `az::SaturatingCast`, float→int via `az`, int→float via `as`, f32→f64 lossless, f64→f32 with explicit clamp to `[f32::MIN, f32::MAX]`).
  - `TryCast<U>` (Tier 3a, strict) keyed on a new element-layer `TryCastElement<U>` trait (mirroring `LossyCastElement<U>`). Primitive pairs are covered by a single blanket impl gated on a sealed `Primitive` marker (delegates to `NumCast::from`); feature-gated storage types provide their own `TryCastElement` impls in their `feat/<type>.rs` modules. Errors as `Error::InvalidElement` on element-cast failure, `Error::InvalidElement` on post-cast non-finite (via `Element::validate`), or `Error::InvalidBoundPair` on cast-induced collision. The sealed-`Primitive`-marker pattern lets the primitive blanket coexist with feat-type impls without coherence conflict — Rust knows feat types can never become `Primitive` downstream.
  - `ordered-float` feature: `LossyCastElement<OrderedFloat<U>> for OrderedFloat<T>` and `LossyCastElement<NotNan<U>> for NotNan<T>` (unwrap → inner element-layer lossy cast → re-wrap). `TryCastElement<OrderedFloat<U>> for OrderedFloat<T>` and `TryCastElement<NotNan<U>> for NotNan<T>` (delegate to ordered-float's upstream `NumCast` impls). `CastElement<NotNan<f64>> for NotNan<f32>` (lossless widening via ordered-float's upstream `From<NotNan<f32>> for NotNan<f64>`); no analogous upstream `From` exists for `OrderedFloat` cross-`T`, so users widen via `TryCast` (always `Ok`).
  - `MaybeDisjoint<T>` cast coverage. Per-variant delegation to the inner `EnumInterval` casts. `Cast` (widening) preserves Disjoint invariants by definition. `TryCast` errors as `Error::InvalidBoundPair` on cast-induced invariant breakage (narrowing collapses one piece to empty, flips order, or makes the pieces touch/overlap). `LossyCast` repairs: empties drop, connecting pieces merge via `MergeConnected`.
  - `fixed` feature: cast coverage for every `FixedI8`/`FixedU8`/`FixedI16`/.../`FixedI128`/`FixedU128` storage type. Two blanket impls per storage type leverage fixed's own `ToFixed`/`FromFixed` conversion traits and `checked_*`/`saturating_*` methods:
    - `TryCastElement<FixedI*<Frac>> for Src where Src: ToFixed` — covers every primitive (int/float/bool) and every other Fixed type in one impl per target. Body delegates to `Fixed::checked_from_num`.
    - `TryCastElement<Dst> for FixedI*<Frac> where Dst: FromFixed + Primitive` — Fixed → primitive narrowing via `Fixed::checked_to_num`.
    - `LossyCastElement<FixedI*<Frac>> for Src where Src: ToFixed` — saturating via `Fixed::saturating_from_num` (handles cross-Fixed widening/narrowing, primitive → Fixed clamp, float → Fixed including `NaN → 0`).
    - `LossyCastElement<Dst> for FixedI*<Frac> where Dst: FromFixed + Bounded + Primitive` — Fixed → primitive saturating via `Fixed::saturating_to_num`.
    - The sealed `Primitive` bound on target-side blankets is what disambiguates cross-Fixed casts (handled exclusively by source-side blankets) from Fixed → primitive casts.
    - No `CastElement` impl: lossless conversion between fixed types is value-dependent across (storage, Frac, value). `fixed`'s own `LosslessTryFrom`/`LossyFrom` traits cover the conversion-typology lattice if downstream needs the exact-conversion semantics.
    - NaN handling: `fixed::checked_from_num(f64::NAN)` returns `None` (matches `TryCast`); `fixed::saturating_from_num(f64::NAN)` returns `0` (matches `LossyCast`'s "total, no Tier 4 panic site" contract — unlike `az`'s int targets).
  - **Coherence fix**: all `LossyCastElement<U> for {feat-type}` blankets now require `U: crate::cast::Primitive`. This prevents coherence overlap when multiple feature-gated storage types are enabled simultaneously (e.g. `rust_decimal` + `fixed` — Rust can't rule out a future `Decimal: ToFixed` or `Fixed: NumCast` impl, so the blankets must be scoped to primitive targets only). Cross-feat-type casts (e.g. `Decimal → BigInt`) are not provided in either direction; users wire them manually if needed.
  - `rust_decimal` feature: cast coverage for `Decimal`.
    - `CastElement<Decimal> for {i8..i128, isize, u8..u128, usize}` via upstream `From<int>` impls — lossless integer widening (Decimal's 96-bit mantissa accommodates every primitive integer).
    - `CastElement<Decimal> for Decimal` reflexive.
    - **No** `CastElement<Decimal> for f*`: unlike `BigDecimal` (unbounded), `Decimal` is bounded (≈ ±7.92e28). A finite f64 like `1e30` overflows Decimal's range, so float→Decimal must be `TryCast`.
    - `TryCastElement<Decimal>` for the same int sources (via `From`, infallible) plus `{f32, f64}` (via upstream `TryFrom<f*>`; fails on NaN/±INF AND on out-of-range finite floats).
    - `TryCastElement<U> for Decimal where U: NumCast + Primitive` — `Decimal → primitive` narrowing via `NumCast::from`.
    - `TryCastElement<Decimal> for Decimal` reflexive.
    - `LossyCastElement<U> for Decimal where U: NumCast + Bounded` — saturating downcast, clamps to `U::min_value()`/`U::max_value()` based on sign.
    - `LossyCast` *targeting* `Decimal` is intentionally not provided: `Decimal` does not impl `num_traits::Bounded` upstream and the orphan rule prevents us from adding it.
  - `num-bigint` feature: full cast coverage for both `BigInt` and `BigUint`.
    - `CastElement<BigInt> for {i8..i128, u8..u128, isize, usize, bool}` via upstream `From<int>` impls — lossless integer widening to arbitrary precision.
    - `CastElement<BigInt> for BigUint` via upstream `From<BigUint> for BigInt` — lossless.
    - `CastElement<BigUint> for {u8..u128, usize, bool}` via upstream `From<u*>` impls.
    - `CastElement<BigInt> for BigInt` and `CastElement<BigUint> for BigUint` reflexive.
    - `TryCastElement<BigInt> for {primitives + BigUint}` via `From` (infallible) and `TryCastElement<BigInt> for {f32, f64}` via `FromPrimitive::from_f*` (truncates fractional like other f→int paths; returns `None` on NaN/±INF).
    - `TryCastElement<BigUint>` for the same source set; signed integer / `BigInt` sources fail on negative values (`TryFrom`-routed).
    - `TryCastElement<U> for BigInt where U: NumCast + Primitive` and `TryCastElement<U> for BigUint where U: NumCast + Primitive` — arbitrary-precision → primitive narrowing via `NumCast::from`.
    - `LossyCastElement<U> for BigInt where U: NumCast + Bounded` — saturating downcast, clamps to `U::min_value()`/`U::max_value()` based on sign.
    - `LossyCastElement<U> for BigUint where U: NumCast + Bounded` — overflow always clamps to `U::max_value()` (BigUint is non-negative).
    - **Floats cannot `Cast` to `BigInt`/`BigUint`** — fractional values would be silently truncated, violating Cast's "lossless" contract. Use `TryCast` (which matches the truncating-fractional behavior of the primitive `TryCast<i32> for f64` blanket).
    - `LossyCast` *targeting* `BigInt`/`BigUint` is intentionally not provided: both are unbounded.
  - `bigdecimal` feature: full cast coverage in both directions.
    - `CastElement<BigDecimal> for {i8..i128, u8..u128}` (via upstream `From<int>` impls, lossless integer widening).
    - `CastElement<BigDecimal> for {f32, f64}` (via `BigDecimal::try_from(...).expect(...)`). The `.expect()` is sound by precondition — the set-level `Cast` impl only ever invokes this on bounds that have passed `Element::validate`, which rejects NaN/±INF for library float types. **This is the headline new capability**: `Interval<f64>::cast::<Interval<BigDecimal>>()` is infallible (no `.unwrap()` at the call site), justified by the `FiniteBound` invariant.
    - `CastElement<BigDecimal> for BigDecimal` reflexive impl.
    - `TryCastElement<BigDecimal> for {i8..i128, u8..u128}` (via `From`, infallible) and `for {f32, f64}` (via `TryFrom`, returns `None` on NaN — for callers operating outside the `FiniteBound` invariant).
    - `TryCastElement<U> for BigDecimal where U: NumCast + Primitive` — `BigDecimal` → primitive narrowing (returns `None` if the value doesn't fit, surfacing as `Error::InvalidElement`).
    - `TryCastElement<BigDecimal> for BigDecimal` reflexive impl.
    - `LossyCastElement<U> for BigDecimal where U: NumCast + Bounded` — saturating downcast from arbitrary-precision to primitive. Out-of-range values clamp to `U::min_value()` / `U::max_value()` based on sign.
    - `LossyCast` *targeting* `BigDecimal` is intentionally not provided: `BigDecimal` is unbounded (no saturation extremum), so the trait's snap-to-extremum semantics have no meaningful interpretation.
- `az` 1.3 added as a default dependency (MIT/Apache, tspiteri) — used internally by the cast module's `LossyCastElement` impls for integer-saturation logic.
- `num_traits::{Bounded, NumCast, ToPrimitive}` re-exported from `numeric`.
- `numeric::Midpointable` (the storage-type trait formerly called `numeric::Midpoint`) is now `pub`. Library impls use `Error = Infallible` for integers / `f32` / `f64` / `OrderedFloat` / `NotNan` / `BigInt` / `BigUint` / `BigDecimal` / `Fixed*`. `Decimal` uses `Error = MathError` (`Range` on rounding overflow at extreme values). The trait name takes the `-able` suffix to free `Midpoint` for the set-level trait, matching the `Width` / `Widthable`, `Cardinality` / `Countable` pattern.
- `ops::Midpoint<T>` trait — set-level midpoint with hull semantics. `fn midpoint(&self) -> Result<T, Self::Error>` with `type Error: core::error::Error;` so wrapper crates can use their own umbrella error type while core uses `MathError`. Impls for `FiniteInterval`, `HalfInterval`, `EnumInterval`, `MaybeDisjoint`. The trait returns the midpoint of the **convex hull** — for a single connected interval, the interval's midpoint; for `MaybeDisjoint::Disjoint(a, b)`, the midpoint of the spanning interval (which **may lie in a gap**, i.e. not be a member of the set). Empty / half-bounded / unbounded inputs return `Err(MathError::Domain)`. The method is named `midpoint`, not `try_midpoint`: there is no panicking sibling — hull-midpoint failure is a routine outcome rather than an exceptional one. Callers needing an in-component point should use `Centroid` (planned).
- `MaybeDisjoint` promoted to a first-class set type in core for operations whose output cardinality stays bounded:
  - **Predicates / queries:** `Contains<X> for MaybeDisjoint` (and commutative `Contains<&MaybeDisjoint> for X`) for `X` ∈ {`&T`, `FiniteOrdBound<&T>`, `&FiniteInterval`, `&HalfInterval`, `&EnumInterval`, `&MaybeDisjoint`}. Self-side checks "any piece contains rhs" (a connected rhs cannot straddle a gap); rhs-side checks "every piece of rhs is contained." `Intersects<&X> for MaybeDisjoint` for the same `X` set plus commutative wrappers — predicates collapse to yes/no, so the cardinality-blowup that rules out `Intersection` does not apply. `Connects<&X> for MaybeDisjoint` (plus commutative) for `X` ∈ {`Finite`, `Half`, `Enum`, `MaybeDisjoint`}; the MD-MD case uses bipartite reuse of the single-piece-rhs formula (`a.connects(rhs) || b.connects(rhs)` in each direction) rather than the overly strict complete-bipartite check.
  - **Producing ops with bounded output:** `MergeConnected` for the same operand grid (output is `Option<single piece>`); pre-checks `connects` before hull-merging, because the hull of `Disjoint(a, b)` includes the gap and a piece in the gap would otherwise hull-merge to a false `Some(...)`. `Split for MaybeDisjoint` with output `(MaybeDisjoint, MaybeDisjoint)` — each side caps at two pieces because at most one fragment per original piece lands per side. `into_hull(self) -> EnumInterval<T>` / `hull(&self)`, `take_min(&mut self)` / `take_max(&mut self)` inherent methods.
  - **Measure:** unified `Measure` impl sums the per-piece measures via `TryAdd`, propagating `Infinite` from either piece and surfacing per-piece or fold overflow as `Err(MathError::Range)`. (`Midpoint` for `MaybeDisjoint` is covered under the `ops::Midpoint` trait entry above — hull-midpoint for `Disjoint(_, _)`.)
  - **Parity:** `Default::default() → MaybeDisjoint::empty()`. `Display` delegates to the inner `EnumInterval` for `Connected(iv)` (so an empty MD prints `{}`); `Disjoint(a, b)` wraps both pieces in set-notation braces with a ` U ` (union) separator: `{[0, 5] U [10, 15]}`. `Zero`/`One`/`SetBounds`/`OrdBounded` are deferred (arithmetic ops blocked by the same cardinality concerns as `Intersection`; bound-trait semantics for multi-piece sets are a separate design question).
  - **Fuzzing / serialization:** `Arbitrary` (arbitrary crate) and quickcheck `Arbitrary` impls generate two `EnumInterval`s and feed them through `MaybeDisjoint::from_pair`, which handles invariant repair (merge if connecting, swap if reversed, collapse if either empty). The natural distribution covers all three observable states without weighted variant selection. Quickcheck dev-dep is enabled at test time via `cfg(any(feature = "quickcheck", test))` so `cargo test` reaches the impls without an explicit feature flag. `Serialize` / `Deserialize` follow the existing `RawMaybeDisjoint<T> + TryFrom` pattern with strict semantics — payloads violating the `Disjoint` invariants (connecting pieces, swapped order, empty piece) are rejected as `Error::InvalidBoundPair` rather than normalized.
- Regression tests pinning `IntervalSet<T>` `Ord` semantics: the derive over `Vec<Interval<T>>` composed with `Interval<T>`'s `OrdBoundPair`-based delegation produces lex-on-bound-sequence ordering — exactly the scheme `MaybeDisjoint` now uses by hand. Six tests cover empty-is-minimum, equal-sets, inner-bound-distinguishability (the `MaybeDisjoint` counterexample lifted to `IntervalSet`), earlier-leftmost-wins, prefix-shorter-loses, and transitivity, so a future change to `Interval` or to the `IntervalSet` field shape surfaces as a test failure rather than a silent shift.
- `measure::Measure` — unified measure trait with `.measure() -> Extent<T::Measure>` (panicking) and `.try_measure() -> Result<Extent<T::Measure>, MathError>` (fallible). For discrete `T` returns the cardinality; for continuous `T` returns the Lebesgue width. Single trait, single call site — the natural measure of `T` flows out of `Element::Measure`. Impls cover `FiniteInterval`, `HalfInterval`, `EnumInterval`, `MaybeDisjoint` (in core) and `Interval`, `IntervalSet` (in the wrapper crate). Coverage includes `f32`/`f64`/`OrderedFloat<F>`/`NotNan<F>`/`Decimal`/`BigDecimal` (continuous: Lebesgue width; singleton intervals report width `0.0`, empty stays `0`) and every primitive integer plus `BigInt`/`BigUint`/`Fixed*` (discrete: cardinality count).
- `numeric::Element` reshape — `Element` gains:
  - `type Kind: sealed::Kind` set to `DiscreteKind` or `ContinuousKind` (sealed marker types) — the category of the element type, used to express the discrete/continuous split at the type level.
  - `type Measure: Zero + TryAdd<Self::Measure, Output = Self::Measure> + Clone` — the natural measure type. Per-T canonical: stepwise widening for primitive integers, self for floats and arbitrary-precision types.
  - `fn try_measure_finite(left: &Self, right: &Self) -> Option<Self::Measure>` — the per-T measure primitive (cardinality `right - left + 1` for discrete, Lebesgue width `right - left` for continuous; `None` ⇒ representation overflow).
- `numeric::DiscreteElement` / `ContinuousElement` — pure-marker subtraits blanket-impl'd over `Element<Kind = DiscreteKind>` / `Element<Kind = ContinuousKind>`. No additional bounds (in particular, **not** `+ Ord` — total order is an orthogonal axis to discreteness). Useful for ergonomic `where`-clause narrowing at call sites.
- Stepwise widening for primitive integer `Element::Measure`: `u8`/`i8 → u16`, `u16`/`i16 → u32`, `u32`/`i32 → u64`, `u64`/`usize`/`i64`/`isize → u128`, `u128`/`i128 → u128` (the 128-bit primitives keep `u128` as their own measure — width fits, cardinality of `[MIN, MAX]` overflows and surfaces as `Err`). The widening keeps arithmetic in native or near-native widths on common targets (~2× faster than `u128` on 32-bit MCUs where 128-bit ops are software-emulated) without giving up overflow safety. Floats stay self-typed. Fixed-point types use `u128` (cardinality count of ULPs).
- `numeric::default_discrete_count_inclusive` — helper function for hand-rolled `Element` impls on user discrete types whose `Measure = Self` and that implement `CheckedSub`. Computes `(right + 1) - left` (or `right - (left - 1)` near the upper limit); returns `None` if both adjacents are unrepresentable. Used internally by the `BigInt` / `BigUint` impls; available for downstream user-type impls.
- `default_continuous_element_impl!` macro (public) — for user continuous types whose `Measure = Self` and whose subtraction is infallible (e.g. arbitrary-precision types like `BigDecimal`). Sets `Kind = ContinuousKind`, `try_adjacent → None`, `try_measure_finite → Some(right - left)`, keeps the default `validate`. For float-like user types that need `is_finite`-based `validate`, hand-roll `Element`.
- `Extent::try_binop_map` — short-circuits on `Infinite`, runs the closure on `Finite + Finite` and propagates its error. Used by the wrapper crate's `IntervalSet::try_measure` summation fold.
- `Extent` derives `Eq`, `Hash`, `Ord` (all T-conditional alongside the existing `PartialEq` / `PartialOrd`). Float `T` continues to lack the conditional impls — same as stdlib.
- `impl From<Extent<T>> for Option<T>` (`Finite(t)` → `Some(t)`, `Infinite` → `None`).
- `impl From<Option<T>> for Extent<T>` (`Some(t)` → `Finite(t)`, `None` → `Infinite`). Canonicalizes the `None` ≡ `Infinite` mapping by fiat; callers must accept it.
- `impl TryAdd for Extent<T>` where `T: TryAdd<T, Output = T>`. `Infinite` short-circuits to `Ok(Infinite)`; `Finite + Finite` delegates to `T::try_add` and propagates `T::TryAdd::Error`. Overflow is **not** collapsed into `Ok(Infinite)` — keeping the three-state (`Ok(Finite)` / `Ok(Infinite)` / `Err`) contract honest. Saturate-on-overflow semantics live in the `core::num::Saturating<T>` storage type.
- `impl Display for Extent<T>` where `T: Display`. `Finite(t)` delegates to `T`'s `Display`; `Infinite` prints as `"∞"`.
- Optional `approx` feature with `AbsDiffEq` / `RelativeEq` / `UlpsEq` impls for `FiniteBound`, `FiniteInterval`, `HalfInterval`, `EnumInterval`, and `MaybeDisjoint` ([#215](https://github.com/gechelberger/intervalsets/pull/215)).

### Changed

- **Breaking:** `Display` for `FiniteInterval` / `HalfInterval` / `EnumInterval` / `MaybeDisjoint` now uses `..` for unbounded sides instead of `<-` / `->`. Half-bounded prints as `[a, ..)` / `(.., b]`; fully unbounded as `(.., ..)`. Aligns with Rust's range idiom (`0..=10`, `0..`, `..10`). The new format is the contract the parallel `FromStr` impl inverts. Migration: update any consumers that pattern-match on the rendered string.
- **Breaking:** `error::Error::InvalidBoundLimit` renamed to `Error::InvalidElement`. The variant's role is unchanged — it's what `Element::validate` rejection (NaN, ±INF, custom predicates) and `TryCmp::try_cmp` incomparability surface as. The new name reads better at call sites and aligns with the `Element` trait that produces it. Migration: rename in any `match` arms (`Error::InvalidBoundLimit` → `Error::InvalidElement`).
- **Breaking:** `measure::Count` / `measure::Cardinality` / `measure::Width` traits collapse into the single `measure::Measure` trait. Per-element capability traits `measure::Countable` and `measure::Widthable` are absorbed into the reshaped `Element` (see Added). Methods `count()` / `try_count()` / `cardinality()` / `try_cardinality()` / `width()` / `try_width()` all collapse to `measure()` / `try_measure()`. The original `Count` method-name conflict with `Iterator::count` is resolved by the rename + collapse.
  - Migration: `s.count()`, `s.cardinality()`, `s.width()` → `s.measure()` (returns the natural measure of `T`).
  - **Continuous singleton semantic change:** `[5.0, 5.0].measure() == Finite(0.0)` (Lebesgue width of a singleton is 0). The pre-collapse `[5.0, 5.0].cardinality() == Finite(1)` (counting-measure interpretation) is dropped — the natural measure on continuous `T` is width, not cardinality.
  - **Discrete output type change:** `[0_i32, 10].measure().finite() == 11_u64` (cardinality `b − a + 1`, widened one bit-class). The pre-collapse `[0_i32, 10].width().finite() == 10_u128` is gone — callers who want `b − a` in the native type should use `ops::Span` (`Span::span` returns `sup − inf` in `T`'s own type).
- **Behavioral break:** Factory methods now reject `±INF` for `f32`/`f64`/`OrderedFloat<f*>`/`NotNan<f*>`. The fallible `try_*` variants return `Err(Error::InvalidElement)`; NaN handling is unchanged but now reports as `InvalidElement` rather than `TotalOrderError` for paths that funnel through the new chokepoint. `ConvertingFactory::Error` now requires `From<Error>` so factory-level convenience methods can propagate validation failures uniformly. All in-tree implementors already satisfy this.
- Implementors of the factory traits now implement `TryFiniteFactory` / `TryHalfBoundedFactory` (the fallible halves) and pick up `FiniteFactory` / `HalfBoundedFactory` for free via blanket impl. External users with custom factory types must rename `fn finite`/`fn half_bounded` overrides to `fn try_finite`/`fn try_half_bounded` and drop the panicking method bodies.
- The factory traits dropped their second type parameter: `FiniteFactory<T, C = Identity>` is now `FiniteFactory<T>`, and likewise for `EmptyFactory`, `HalfBoundedFactory`, `UnboundedFactory`, `TryFiniteFactory`, `TryHalfBoundedFactory`. Implementors drop the trailing `, Identity` parameter from their impls.
- The shared base trait that declares `Output` / `Error` was renamed `ConvertingFactory` → `Factory` (the "Converting" prefix referred to the now-removed `Converter` trait).
- **Behavioral break:** Set-level math (`TryAdd` / `TrySub` / `TryMul` / `TryDiv` for `FiniteInterval` / `HalfInterval` / `EnumInterval` and the heterogeneous combinations) is re-bound from `T: Add<Output = T>` (etc.) to `T: TryAdd<Output = T>` (etc.). Bound math now propagates value-level overflow / non-finite / divide-by-zero through `T::TryAdd::Error` into `Error::Math(MathError)`. For primitive integers this means overflow surfaces as `Err(MathError::Range)` instead of silently wrapping in release; for primitive floats, `INF` / `NaN` results surface as `Err(MathError::Domain)`. The `iN::MIN / -1` panic that previously slipped through `Div` for signed integers in release mode is now caught here and reported as `Err(MathError::Range)`.
- **Behavioral break:** Infix `+` / `-` / `*` / `/` on set types (`FiniteInterval` / `HalfInterval` / `EnumInterval`) is now explicit panicking sugar over `try_op().unwrap()`. The previous `T: Ord`-backed "provably infallible" framing is dropped; the bound becomes `Self: TryAdd<Output = Self>` plus `<Self as TryAdd>::Error: Debug` (analogous for sub/mul/div). Infix **may panic in release** when the corresponding `try_*` returns `Err`. The panic site is part of the documented contract.
- `FiniteBound<T>` infix `Add` / `Sub` / `Mul` re-bound the same way: the bound flips to `T: TryAdd<Output = T>` (etc.) and the impl bodies become `try_op().unwrap()`. The output type tightens from `FiniteBound<<T as Add>::Output>` to `FiniteBound<T>` because `T: TryAdd<Output = T>` locks the inner output. `Zero` / `One` / `ConstZero` / `ConstOne` impls on `FiniteBound` and on `FiniteInterval` / `EnumInterval` pick up the new `Try*` bound chain transitively.
- Tier 3 documentation in `ops/mod.rs` is split into **Tier 3a** (`try_*`: total, panic-free in release, canary-verified) and **Tier 3b** (infix sugar: may panic in release and debug). The `ops/math/mod.rs` "Panicking and fallible forms" section is rewritten to drop the `T: Ord`-makes-unwrap-safe claim and frame infix as honest panicking sugar.
- `IntervalSet::try_measure` summation surfaces representation overflow via a `TryAdd`-based fold rather than panicking in debug / wrapping in release. The bound chain is `T: Element` (which provides `T::Measure: Zero + TryAdd + Clone` transitively) plus `<T::Measure as TryAdd>::Error: Into<MathError>`. Per-piece `Infinite` propagates to an infinite total; per-piece or fold overflow surfaces as `Err(MathError::Range)`.
- Rename `ops::IntoFinite` → `ops::IntoFiniteInterval` and its method `into_finite()` → `into_finite_interval()`. The trait contract and the `FiniteInterval` / `HalfInterval` / `EnumInterval` impls are unchanged; the rename clarifies that the operation collapses to a single finite interval rather than a "set of finite elements." Migration: rename references at the call site.
- Rename `measure::Measurement<T>` → `measure::Extent<T>` and move it from `measure/mod.rs` to its own `measure/extent.rs` submodule. The variant shape (`Finite(T)` / `Infinite`) is unchanged. Migration: rename `Measurement` → `Extent` at the call site. **Note** the inherent-method trim below — the kept `finite` / `is_finite` / `is_infinite` / `try_binop_map` cover their previous semantics; callers using `expect_finite` / `finite_or` / `flat_map` / `map` route through `Option::from(extent)` and the std `Option` combinators.
- Float `Midpoint` impls (`f32`, `f64`, `OrderedFloat`, `NotNan`) drop the runtime non-finite check now that `Element::validate` rejects non-finite at the `FiniteBound` chokepoint (see #241). Direct callers passing non-finite values bypass the in-tree contract and inherit std's `f*::midpoint` semantics (typically a NaN-tainted result rather than `Err`).
- **Behavioral break:** `MaybeDisjoint` `Ord` / `PartialOrd` are now hand-written as lex-on-pieces over the bound sequence (the k-piece generalization of `OrdBoundPair`'s order). The previous `derive` ordered by enum tag first (`Connected` < `Disjoint`), which collapsed `Disjoint([a,b],[c,d])` vs `Disjoint([a,y],[z,b])` pairs that share an outer hull but differ on inner bounds — those are now ordered by the first differing bound. `Connected(EnumInterval::empty())` continues to sort below every inhabited value (it inherits `OrdBoundPair::empty()`'s minimum status, no special-case needed).
- **Breaking:** `MaybeDisjoint` now implements `IntoIterator` instead of `Iterator`. `Iterator for MaybeDisjoint` was a method-resolution footgun — `md.cmp(&other)`, `md.partial_cmp(&other)`, `md.lt(&other)` all resolved to the `Iterator` versions (consuming `self`, operating over yielded items) rather than the `Ord` / `PartialOrd` trait impls. Replacement is `MaybeDisjoint: IntoIterator` with a dedicated `MaybeDisjointIntoIter` (plus `FusedIterator`). Migration: `for piece in md`, `md.into_iter()`, and combinator chains still work; bare `md.next()` / `md.map()` / etc. now require an explicit `.into_iter()`.
- **Breaking:** `MaybeDisjoint` moved from `crate::disjoint::MaybeDisjoint` to `crate::sets::MaybeDisjoint`. The standalone `disjoint` module is gone and `sets.rs` is split into `sets/{finite, half, enum_interval, disjoint}.rs`. Re-import paths accordingly.
- **Breaking:** `impl<T> From<FiniteOrdBound<T>> for FiniteBound<T>` replaced with `impl<T: Element> TryFrom<FiniteOrdBound<T>> for FiniteBound<T>`. The previous `From` did not run `Element::validate` and could silently produce a `FiniteBound` carrying a value the type's invariant disallows (e.g. `f64::INFINITY`). The conversion now validates at the boundary; `TryFrom<OrdBoundPair<T>> for EnumInterval<T>` propagates via `?`. See the Fixed section for the soundness story.
- **Breaking:** `BoundType::combine` renamed to `BoundType::meet` (the open/closed lattice meet — result is `Closed` iff both inputs are `Closed`). Doc-string added. Both `BoundType::meet` and `BoundType::flip` are now `const fn`. The unused enum discriminant assignments (`Open = 0`, `Closed = 1`) are dropped — no `#[repr]` exists for them to be load-bearing.
- **Breaking:** `FiniteBound::contains_bound_assume_valid` and `try_contains_bound` take `FiniteBound<&T>` instead of `FiniteOrdBound<&T>`. The `finite_ord(side)` conversion moves inside; callers pass `b.as_ref()` instead of `b.finite_ord(side)`.
- `OrdBoundPair::is_empty` no longer requires `T: PartialEq`. Detection uses a discriminant `matches!` instead of an equality check against the canonical empty marker.
- Standardized "contains" wording across `FiniteBound::contains_assume_valid` / `try_contains` / `contains_bound_assume_valid` / `try_contains_bound` docs (previously mixed "contains" and "sees").

### Removed

- **Removed** the `Converter` trait, the `Identity` converter, the `EIFactory<T, C>` type-level factory, and `ConvertingFactory::try_convert`. The trait was a tutorial-quality nicety for end users wanting to construct `OrderedFloat`/`NotNan`-wrapped intervals from raw `f32`/`f64` values; nothing internal used it. Migration: wrap the value directly at the call site — `EnumInterval::closed(NotNan::new(0.0).unwrap(), NotNan::new(10.0).unwrap())` or define a project-local helper. `OrderedFloat::from(value: T)` exists for the infallible case.
- **Removed** the `Error::TotalOrderError(TotalOrderError)` variant from the umbrella enum. `From<TotalOrderError> for Error` now collapses to `Error::InvalidElement` — same destination as `Element::validate` rejection, since both gates fire on the same root cause (a bound's value isn't a usable limit). The `TotalOrderError` struct itself is unchanged and remains the precise return type of `TryCmp::try_cmp`. Migration: replace `Err(Error::TotalOrderError(_))` matchers with `Err(Error::InvalidElement)`.
- **Removed** `error::MidpointError` (was `pub(crate)`; never user-visible). External `Midpointable` impls that aliased it should adopt `MathError` (or `Infallible`) for `type Error`.
- **Removed** `Extent::Sub` impl. The previous impl returned `Infinite` whenever either operand was `Infinite`, which is mathematically wrong (`Finite(n) - Infinite` should be `-Infinite`; `Infinite - Infinite` is undefined). No in-repo callers; downstream consumers can compose with `Extent::try_binop_map` plus a caller-supplied checked subtraction.
- **Removed** unused `Extent::binop_try_map` helper (was `#[allow(dead_code)]`).
- **Removed** dead `T: Clone` bound on `Extent::Add` (the `binop_map` it dispatches through consumes by value).
- **Removed** `Extent::expect_finite`, `Extent::finite_or`, `Extent::map`, `Extent::flat_map` inherent methods — duplicates of `Option`'s `expect` / `unwrap_or` / `map` / `and_then`. Migration: round-trip via the new `From<Extent<T>> for Option<T>` and use the std `Option` combinators, e.g. `Option::from(x).expect(msg)`, `Option::from(x).unwrap_or(default)`, `Option::from(x).map(f).into()`. `finite` / `is_finite` / `is_infinite` / `try_binop_map` are unchanged.
- **Removed:** `measure::Cardinality`, `measure::Width`, `measure::Countable`, `measure::Widthable` traits (and their `.cardinality()` / `.try_cardinality()` / `.width()` / `.try_width()` methods). Collapsed into the unified `measure::Measure` trait — see the Added section and the Changed-section migration table.
- **Removed:** `default_countable_impl!`, `default_width_impl!`, `continuous_countable_impl!`, `continuous_domain_impl!`, `integer_domain_impl!`, `float_domain_impl!` macros. User continuous types use the new public `default_continuous_element_impl!` macro; user discrete types hand-roll `Element` (with the new `default_discrete_count_inclusive` helper for the cookie-cutter case). Primitive integer / float impls are baked into `element.rs` directly via private macros.
- **Removed:** `Countable::IS_CONTINUOUS` flag. The discrete/continuous distinction now lives in `Element::Kind` (a type-level marker), not in a per-impl const.
- **Removed:** `numeric::CheckedSub` re-export's load-bearing role for `default_countable_impl!`. The trait stays re-exported (still used by `default_discrete_count_inclusive`), but the old macro that consumed it is gone.
- **Breaking:** removed `MaybeDisjoint::Consumed`. The empty set is now canonically `Connected(EnumInterval::empty())`, which also serves as the drained-iterator sentinel — eliminating the dual identity where `Consumed` and `Connected(empty)` denoted the same set but differed at the type level. Migration: replace `MaybeDisjoint::Consumed` patterns with `MaybeDisjoint::empty()` / `Connected(EnumInterval::empty())`, and reach for `MaybeDisjoint::is_inhabited()` rather than matching on the variant.
- **Breaking:** the public constructor surface of `FiniteOrdBound` and `OrdBound` is gone. `FiniteOrdBound::{new, closed, open}` and `OrdBound::{new_finite, closed}` are now `pub(crate)` and renamed with the `_assume_valid` suffix to make their Tier-4 (no-`Element`-validation) status explicit. Dead `OrdBound::{left_open, right_open}` constructors are deleted. External users obtain `FiniteOrdBound`/`OrdBound` only by destructuring an `OrdBoundPair` returned from `From<&Interval>` (or sibling impls) — these types are conversion currency, not a primary construction surface.
- **Breaking:** `numeric::element`, `numeric::midpoint`, and `numeric::saturating` are no longer `pub mod`. Their public items (`Element`, `DiscreteKind`, `ContinuousKind`, `DiscreteElement`, `ContinuousElement`, `default_discrete_count_inclusive`, `Midpointable`) remain reachable through the existing `numeric::*` re-exports; `numeric::saturating` had no public items. Migration: drop the inner module segment from any `numeric::element::*` / `numeric::midpoint::*` imports.

### Fixed

- `Element::validate` for `Decimal` normalizes the limit to its canonical `(mantissa, scale)` form via `Decimal::normalize`. Previously two intervals built from value-equal but scale-distinct inputs (e.g. `Decimal::new(10, 0)` vs `Decimal::new(100, 1)`) stored different bit patterns despite denoting the same set, leaking through any downstream consumer that distinguished bound representations rather than values (debug output, hashing of bound bytes, etc.).
- Measure on integer intervals no longer panics in debug / produces a *negative width* in release on `[i32::MIN, i32::MAX]` and similar full-range cases. Each primitive integer `Element::Measure` widens one bit-class so the diff always fits, and `try_measure` surfaces 128-bit `[MIN, MAX]` cases as `Err(MathError::Range)`.
- `IntervalSet::try_measure` summation is now panic-free at every step — previously the per-interval measure was checked but the fold-step `Add` was not, so a set whose individual measures fit but whose sum did not would panic in debug / wrap in release.
- `measure` ops (`try_measure`, `midpoint`) are now covered by the panic-free canary (`tier2_measure`).
- **Soundness:** the documented `OrdBoundPair → EnumInterval` round-trip on `f32`/`f64` no longer produces an `EnumInterval` carrying a `FiniteBound` with `±INF`. `OrdBoundPair::try_new`'s ordering check is `PartialOrd`-only, which accepts `±INF`; an INF-valued pair could therefore pass `try_new` and slip into a `FiniteBound` via the previous infallible `From<FiniteOrdBound> for FiniteBound`. Validation now fires at the new `TryFrom<FiniteOrdBound<T>> for FiniteBound<T>` chokepoint and surfaces as `Err(Error::InvalidBoundLimit)`.
