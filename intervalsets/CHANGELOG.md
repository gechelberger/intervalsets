## [Unreleased]

### Added

- `interval!` and `set!` proc-macros â€” compile-time-checked literals for `Interval<T>` and `IntervalSet<T>`. Re-exported from the crate root and `prelude`. The new `intervalsets-macros` workspace member supplies the proc-macro implementation; depend on `intervalsets` and the macros are available transparently.
  - `interval!("[0, 10)")` accepts the same grammar as `Interval`'s new `FromStr` impl; round-trips with `Display`. Malformed input (bracket mismatch, missing comma, closed bracket on an unbounded side, set notation, crossed numeric-literal bounds) fails to **build** instead of panicking at runtime. Bound bodies are tokenized as Rust source so non-literal bounds (identifiers, function calls like `BigDecimal::from(0)`) work alongside numeric literals.
  - `set!("{[0, 5] U [10, 15]}", i32)` is the multi-piece analogue. Grammar matches the new `IntervalSet` Display: `{}` for empty, `{piece}` for single, `{piece U piece U ...}` for multi-piece with ASCII ` U ` separator. Pieces don't need to be sorted, non-overlapping, or non-empty in source â€” the macro expands to a chain of `Interval::ctor(...).union(...)` calls and `IntervalSet`'s union machinery normalizes the result.
  - Both macros accept an optional second argument supplying a storage-type hint. The hint is emitted as a turbofish on the constructor (`interval!("(.., ..)", i32)`, `set!("{}", f64)`) and additionally wraps every bound expression in `<T as From<_>>::from(...)`, so any `From`-convertible source may be used as a bound â€” including user newtypes like `struct Unit(f64); impl From<f64> for Unit`. UFCS is used so an inherent `from` on the user's type can't shadow the trait method. For `set!` the hint is applied to every piece (not just the first), so multi-piece literals with heterogeneous bound sources work uniformly. `_` (the placeholder type) is accepted and is equivalent to omitting the hint.
- `FromStr` impl for `Interval<T>` (delegates to the new `EnumInterval` parser in `intervalsets-core`). Round-trips against `Display`. `error::ParseIntervalError` is re-exported from core for users who want to match on parse failures.
- `FromStr` impl for `IntervalSet<T>` â€” parses the new `Display` form: `{}`, `{piece}`, or `{piece U piece U ...}`. Each piece parses via `Interval`'s `FromStr`; the overall result is normalized via `IntervalSet::new` (sort, merge connected, drop empties), so pieces need not be sorted, non-overlapping, or non-empty in the input. Whitespace around delimiters and the ` U ` separator is lenient; leading, trailing, or doubled `U` errors as `ParseIntervalError::Syntax`. Bound: `T: Element + FromStr`.
- `ops::Bisect<T>` impls for `Interval<T>` and `IntervalSet<T>` (trait + `Bisection<T, S>` re-exported from `intervalsets-core`). Single method `bisect(closed)` â€” bisection uses the natural `Measure` of `T` (cardinality on discrete, Lebesgue width on continuous). `Interval` splits at the bound midpoint in one step (uniform measure density). `IntervalSet` walks the convex hull's value range, splitting at the bracket midpoint and walking toward the heavier half until stable. Both return `None` for empty / unbounded inputs.
- `core::num::Saturating<T>` is now a valid storage type (inherited from `intervalsets-core`). All four `Try*` ops are uniformly `Error = Infallible` â€” saturation is defined behavior on the bounded integer lattice, not overflow. `core::num::Wrapping<T>` is intentionally not supported (ordering-flipping overflow violates the bound-pair invariant). See the `intervalsets-core` changelog for the full surface, `TryDiv` saturation rule, and orphan-rule limitation around `IntoFiniteInterval`.
- `cast::{Cast, LossyCast, TryCast}` impls for `Interval` and `IntervalSet` (re-exported via prelude). `Interval` delegates to its inner `EnumInterval`. `IntervalSet::cast` routes through `try_new` (strict; widenings preserve invariants); `IntervalSet::lossy_cast` routes through `new` (repairing â€” narrowed intervals that collapse onto the same range merge); `IntervalSet::try_cast` routes through `try_new` and surfaces cast-induced invariant violations as `Error::InvalidIntervalSet`. Coverage extends to `ordered-float`'s `OrderedFloat<T>` and `NotNan<T>` storage types via the impls landed in `intervalsets-core`.
- `ops::Midpoint<T>` impls for `Interval<T>` and `IntervalSet<T>` (trait re-exported from `intervalsets_core::ops`). Hull-midpoint semantics: `(inf(S) + sup(S)) / 2`. For a single connected `Interval` this is the interval's midpoint; for a multi-piece `IntervalSet` it is the midpoint of the convex hull and **may lie in a gap** between components â€” see the trait docs. Empty / half-bounded / unbounded inputs return `Err(Error::Math(MathError::Domain))`. Wrapper impls use `type Error = crate::error::Error` so the wrapper's umbrella error type carries the result.
- `measure::Measure` impls for `Interval<T>` and `IntervalSet<T>` (trait re-exported from `intervalsets-core`). Unified measure surface â€” `.measure()` returns `Extent<T::Measure>`: cardinality on discrete `T`, Lebesgue width on continuous `T`. The `IntervalSet` impl sums per-piece measures via `TryAdd` and surfaces mid-fold overflow as `Err(MathError::Range)`. Continuous coverage now includes singleton intervals as width `0` (the pre-unification "singleton cardinality = 1" semantic is dropped â€” natural measure on continuous `T` is width). Discrete coverage extends to `OrderedFloat` / `NotNan` wrappers transparently (they keep their inner float's continuous kind).
- Re-export of `numeric::Midpointable` (the storage-type trait, renamed from `numeric::Midpoint` in core). The set-level `Measure` impls for `Interval` / `IntervalSet` use `type Error = MathError`.
- Optional `approx` feature with `AbsDiffEq` / `RelativeEq` / `UlpsEq` impls for `Interval` and `IntervalSet` ([#215](https://github.com/gechelberger/intervalsets/pull/215)).
- `error::Error::InvalidElement` variant lifted from `intervalsets-core` for the new `Element::validate` rejection path.
- `error::Error::Math(MathError)` variant (with `From<MathError> for Error`) so set-level `try_*` math can surface value-level arithmetic failures. `MathError` is re-exported from `intervalsets_core::error`.

### Changed

- **Breaking:** `Display for IntervalSet<T>` now uses ` U ` (ASCII union) as the multi-piece separator instead of `, `, matching the new `set!` macro / `FromStr` grammar. `{[0, 5], [10, 15]}` â†’ `{[0, 5] U [10, 15]}`. Empty (`{}`) and single-piece (`{[0, 10]}`) forms are unchanged. The ` U ` form is visually distinct from bound-internal commas and reads as the union semantics it actually represents; pre-alpha breaking, intentional. Migration: update any string-match consumers of `IntervalSet` Display output. (`MaybeDisjoint` in `intervalsets-core` got the same treatment for consistency â€” see its changelog.)
- **Breaking (inherited from `intervalsets-core`):** `Display` now uses `..` for unbounded sides instead of `<-` / `->` (`[a, ..)` / `(.., b]` / `(.., ..)`). Aligns with Rust's range idiom and pairs with the new `FromStr` impl. Migration: update any string-match consumers.
- **Breaking (inherited from `intervalsets-core`):** `error::Error::InvalidBoundLimit` renamed to `Error::InvalidElement`. Same semantics (Element::validate / partial_cmp rejection), clearer name. Migration: rename in `match` arms.
- **Breaking (inherited from `intervalsets-core`):** `measure::Count` / `measure::Cardinality` / `measure::Width` traits collapse into the unified `measure::Measure` trait. Per-T capability traits `measure::Countable` and `measure::Widthable` are absorbed into the reshaped `Element` (which now carries `type Kind`, `type Measure`, and `try_measure_finite`). Methods `count()` / `cardinality()` / `width()` and their `try_*` siblings all collapse to `measure()` / `try_measure()`.
  - Migration: `s.count()`, `s.cardinality()`, `s.width()` â†’ `s.measure()`.
  - **Continuous singleton semantic change:** `Interval::closed(5.0_f64, 5.0).measure() == Finite(0.0)` (Lebesgue width). Pre-collapse `.cardinality() == 1` (counting-measure interpretation) is dropped.
  - **Discrete output type change:** `Interval::closed(0_i32, 10).measure().finite() == 11_u64` (cardinality, stepwise-widened to `u64`). Previously `.width().finite() == 10_u128` â€” callers wanting `b âˆ’ a` in the native type should use `ops::Span` (returns `sup âˆ’ inf` in `T`'s own type).
- **Behavioral break (inherited from `intervalsets-core`):** `Interval` / `IntervalSet` factory paths now reject `Â±INF` and `NaN` at construction for `f32`/`f64`/`OrderedFloat<f*>`/`NotNan<f*>`. Previously `Â±INF` silently produced a non-canonical interval. Code that constructed intervals from arbitrary floats should filter via `is_finite()` before construction or handle `Err(Error::InvalidElement)` from the `try_*` variants.
- **Behavioral break (inherited from `intervalsets-core`):** Set-level math on `Interval` / `IntervalSet` is re-bound from `T: Add<Output = T>` (etc.) to `T: TryAdd<Output = T>` (etc.) via the wrapped `EnumInterval`. `try_add` / `try_sub` / `try_mul` / `try_div` now propagate value-level overflow / non-finite / divide-by-zero into `Err(Error::Math(MathError::Range | Domain))`. Infix `+` / `-` / `*` / `/` is panicking sugar over `try_op().unwrap()` and **may panic in release** when the underlying `try_*` would have returned `Err` â€” including integer overflow, signed `iN::MIN / -1`, and float `INF` / `NaN` results. The previous `T: Ord` requirement on infix is dropped; the bound now requires that the wrapped `EnumInterval`'s `Try*` impl is available with `Error: Debug`. `Zero` / `One` impls on `Interval` / `IntervalSet` pick up the new `Self: Add<Self, Output = Self>` / `Self: Mul<Self, Output = Self>` chain instead of the previous `T: Ord` requirement.
- **Behavioral break (inherited from `intervalsets-core`):** primitive integer `Element::Measure` uses stepwise widening (`u8`/`i8 â†’ u16`, â€¦, `u64`/`usize`/`i64`/`isize â†’ u128`, `u128`/`i128 â†’ u128`). The widening fixes the previous `[i32::MIN, i32::MAX]` debug-panic / release-wrap, keeps arithmetic in native or near-native widths on common targets, and only the 128-bit `[MIN, MAX]` cardinality overflows (surfaces as `Err(MathError::Range)`).
- **Behavioral break:** `ops::IntoFinite` â†’ `ops::IntoFiniteInterval` and `into_finite()` â†’ `into_finite_interval()` (inherited from `intervalsets-core`). The `IntervalSet<T>` impl additionally changes its `Output` from `Self` to `Interval<T>` and its semantics from per-subinterval clamping to "hull of the set, then truncate to type extents" â€” the trait name promised a single finite interval but the previous impl returned a multi-piece `IntervalSet`. Migration: callers wanting the previous per-piece behavior should collect manually â€” `set.into_iter().map(IntoFiniteInterval::into_finite_interval).filter(MaybeEmpty::is_inhabited).collect::<IntervalSet<_>>()`. The new impl consumes via `OrdBoundPair::from(self)`, so no `T: Clone` bound.
- **Behavioral break (inherited from `intervalsets-core`):** `measure::Measurement<T>` â†’ `measure::Extent<T>` (now living in its own `measure/extent.rs` submodule). Variant shape (`Finite(T)` / `Infinite`) unchanged. The inherent-method surface is trimmed â€” `finite`, `is_finite`, `is_infinite`, `try_binop_map` are kept; `expect_finite`, `finite_or`, `map`, `flat_map` are dropped in favor of routing through the new `From<Extent<T>> â†” Option<T>` impls (use `Option::from(x).expect(msg)`, etc.). `TryAdd` and `Display` impls are now provided on `Extent<T>`. Migration: rename `Measurement` â†’ `Extent` at the call site and substitute the `Option` combinators for the dropped methods.
- **Behavioral break (inherited from `intervalsets-core`):** `TryFrom<OrdBoundPair<T>> for Interval<T>` and `TryFrom<OrdBoundPair<T>> for IntervalSet<T>` now reject inner bound values that fail `Element::validate` (e.g. `f64::INFINITY`). Previously such values slipped through `OrdBoundPair::try_new`'s `PartialOrd`-only ordering check and into the inner `FiniteBound` silently. Migration: callers reconstructing intervals from a hand-built or hand-mutated `OrdBoundPair` should handle `Err(Error::InvalidBoundLimit)` from the round-trip `TryFrom` path. Round-trips that start from a validated `Interval`/`IntervalSet` are unaffected.

### Deprecated

### Removed

- **Removed** `IFactory<T, C>` and `ISFactory<T, C>` (the parameterized factory marker types), along with the underlying `Converter` machinery in `intervalsets-core`. Migration: construct wrapped types directly at the call site â€” `Interval::closed(NotNan::new(0.0).unwrap(), NotNan::new(10.0).unwrap())` or `Interval::closed(OrderedFloat::from(0.0), OrderedFloat::from(10.0))`.
- **Removed** the `Error::TotalOrderError(TotalOrderError)` variant from the umbrella `Error`. `From<TotalOrderError> for Error` now collapses to `Error::InvalidElement`. The `TotalOrderError` struct re-export is unchanged. Migration: replace `Err(Error::TotalOrderError(_))` matchers with `Err(Error::InvalidElement)`.
- **Removed** `Extent::Sub` impl (inherited from core). The previous impl returned `Infinite` whenever either operand was `Infinite`, which is mathematically wrong.
- **Removed** `Extent::expect_finite`, `Extent::finite_or`, `Extent::map`, `Extent::flat_map` inherent methods (inherited from core). Migration: use `Option::from(extent)` and the std `Option` combinators.
- **Removed (inherited from `intervalsets-core`):** `measure::Cardinality`, `measure::Width`, `measure::Countable`, `measure::Widthable` traits and the `.cardinality()` / `.try_cardinality()` / `.width()` / `.try_width()` methods. Collapsed into the unified `measure::Measure` trait.
- **Removed (inherited from `intervalsets-core`):** `default_countable_impl!`, `default_width_impl!`, `continuous_countable_impl!`, `continuous_domain_impl!` macros. User continuous types use the new `default_continuous_element_impl!` macro; user discrete types hand-roll `Element` (with the new `numeric::default_discrete_count_inclusive` helper for the cookie-cutter case).

### Fixed

- `Decimal` bound limits are now normalized at construction, so intervals built from value-equal but scale-distinct inputs (e.g. `Decimal::new(10, 0)` vs `Decimal::new(100, 1)`) share one canonical bit pattern.
- `IntervalSet::try_measure` no longer panics in debug / wraps in release on integer intervals; the per-step `Sub` overflow on full-range integer intervals is gone (stepwise widening to the next unsigned bit-class), and the summation step uses `TryAdd` so a sum that exceeds `T::Measure` surfaces as `Err(MathError::Range)`.
